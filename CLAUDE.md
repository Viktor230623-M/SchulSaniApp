## Architecture

### API Server (`artifacts/api-server`)

- Express.js with JWT auth (24h, stored in cookies or Bearer header)
- Rate limiting: 100 req/min general, 5 req/min on auth endpoints
- Route protection via `authenticateToken` middleware + role checks inline per route
- Business logic lives in `src/services/`; routes are thin controllers
- In-memory data stores in `src/data/` supplement the PostgreSQL DB

### Mobile App (`artifacts/paramedic-app`)

- Expo Router file-based routing under `app/`; tabs live in `app/(tabs)/`
- Admin-only screens under `app/admin/`
- State: Zustand (`store/`) for global app state; TanStack React Query for server state
- API calls go through the generated client in `lib/api-client-react/`
- Tailwind CSS 4 for styling

### Database (`lib/db`)

Drizzle ORM on PostgreSQL. Key tables:

| Table | Purpose |
|---|---|
| `users` | Accounts; roles: `student_paramedic`, `sanitaeter`, `sanitaeter_leitung`, `sanitaeter_leitung_admin`, `admin` |
| `missions` | Emergency calls with status lifecycle |
| `news` | Articles with approval workflow |
| `loa` | Leave-of-absence requests with approval |
| `duty` | Per-user duty status (`on_duty`, `off_duty`, â€¦) |
| `notifications` | Push notifications with read status |
| `device_tokens` | iOS/Android/Web push tokens |
| `mission_activity_log` | Audit trail for mission actions |

Schema source of truth: `lib/db/src/schema/index.ts`. Apply changes with `pnpm --filter db push`.
